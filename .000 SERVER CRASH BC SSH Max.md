# Server Crash / SSH Max Authentication Recovery

Use this when the server is back up but Windows SSH/SCP fails with:

```text
maximum authentication attempts exceeded for michelle-work from 192.168.1.69
```

This usually means Windows/OpenSSH tried too many saved keys before password authentication. It does not prove a hack by itself.

## 1. Windows CMD: Stop Hung SSH/SCP

```bat
taskkill /IM ssh.exe /F
taskkill /IM scp.exe /F
```

## 2. Windows CMD: Test Password-Only SSH

```bat
ssh -o PreferredAuthentications=password -o PubkeyAuthentication=no -o NumberOfPasswordPrompts=3 michelle-work@192.168.1.70
```

If it connects, type:

```bash
exit
```

## 3. Windows CMD: Upload With Password-Only SCP

```bat
cd /d F:\

scp -o PreferredAuthentications=password -o PubkeyAuthentication=no -o NumberOfPasswordPrompts=3 customerportal-upload.tgz michelle-work@192.168.1.70:/home/michelle-work/uploads/customerportal-upload.tgz
```

Then continue the normal deploy steps.

## 4. If It Still Says Max Authentication Attempts

Use the stronger password-only command:

```bat
ssh -o IdentitiesOnly=yes -o IdentityFile=NUL -o PreferredAuthentications=password -o PubkeyAuthentication=no michelle-work@192.168.1.70
```

And upload with:

```bat
scp -o IdentitiesOnly=yes -o IdentityFile=NUL -o PreferredAuthentications=password -o PubkeyAuthentication=no customerportal-upload.tgz michelle-work@192.168.1.70:/home/michelle-work/uploads/customerportal-upload.tgz
```

## 5. Server Terminal Checks If SSH Still Will Not Connect

Run these directly on the server:

```bash
hostname -I
sudo systemctl status ssh --no-pager
sudo ss -ltnp | grep ':22' || true
sudo ufw status verbose
df -h
uptime
```

If SSH is down:

```bash
sudo systemctl restart ssh
sudo systemctl status ssh --no-pager
sudo ss -ltnp | grep ':22' || true
```

## 6. Check For Ban Or Attack Signs

Run on the server:

```bash
sudo journalctl -u ssh -n 80 --no-pager
last -a | head -30
sudo fail2ban-client status sshd 2>/dev/null || echo "fail2ban sshd not active or not installed"
```

If fail2ban shows the Windows IP is banned:

```bash
sudo fail2ban-client set sshd unbanip 192.168.1.69
```

## 7. Do Not Do This

Do not keep rerunning normal `ssh` or `scp` if it repeatedly says max authentication attempts. Stop the hung processes first, then use the password-only commands above.
