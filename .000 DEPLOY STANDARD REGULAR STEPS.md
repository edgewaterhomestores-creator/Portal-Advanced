# Standard Regular Deploy Steps

Use this format when the deploy package already exists as:

`F:\customerportal-upload.tgz`

This is the preferred regular deploy layout because it is grouped, numbered, and not bloated.

## What Usually Changes

- Package number and SHA256.
- Extra `ls` checks only when a new critical file was changed.
- Service names only if a different portal is being deployed.
- Domains only if the public route changes.
- Exclude rules only if a new live-data folder must be preserved.

Do not change the shell, upload method, restart method, or deployment flow unless Michelle explicitly approves it.

## 1. Windows CMD

```bat
cd /d F:\
scp customerportal-upload.tgz michelle-work@192.168.1.70:/home/michelle-work/uploads/customerportal-upload.tgz
ssh michelle-work@192.168.1.70
```

## 2. Linux Server

```bash
rm -rf ~/uploads/customerportal
mkdir -p ~/uploads/customerportal
tar -xzf ~/uploads/customerportal-upload.tgz -C ~/uploads/customerportal

ls ~/uploads/customerportal/package.json
ls ~/uploads/customerportal/package-lock.json
ls ~/uploads/customerportal/server/index.js
ls ~/uploads/customerportal/server/db.js
ls ~/uploads/customerportal/public/sign.html
ls ~/uploads/customerportal/InstallerPortal/package.json
ls ~/uploads/customerportal/InstallerPortal/server.js
```

## 3. Linux Server

```bash
sudo rsync -av --delete --exclude InstallerPortal --exclude node_modules --exclude .env --exclude 'data/generated' --exclude 'data/packets' --exclude 'data/logs' --exclude 'data/settings' --exclude 'data/estimates' --exclude 'data/estimate-module' --exclude 'data/preimport' --exclude 'data/quick-contracts' ~/uploads/customerportal/ /opt/apps/customerportal/app/

sudo rsync -av --delete --exclude node_modules --exclude .env --exclude 'data/installers/installer-job-photos' --exclude 'data/notifications' --exclude 'data/*.log' ~/uploads/customerportal/InstallerPortal/ /opt/apps/installerportal/app/

sudo chown -R customerportal:customerportal /opt/apps/customerportal/app
sudo chown -R installerportal:installerportal /opt/apps/installerportal/app
```

## 4. Linux Server

```bash
cd /opt/apps/customerportal/app
sudo -u customerportal npm ci --omit=dev

cd /opt/apps/installerportal/app
sudo -u installerportal npm ci --omit=dev

sudo systemctl restart customerportal
sudo systemctl restart installerportal
sleep 5

sudo systemctl status customerportal --no-pager
sudo systemctl status installerportal --no-pager

curl -fsS http://127.0.0.1:3000/api/health; echo
curl -fsS http://127.0.0.1:3011/api/health; echo
curl -k --resolve contracts.edgefam.com:443:127.0.0.1 https://contracts.edgefam.com/api/health; echo
curl -k --resolve installers.edgefam.com:443:127.0.0.1 https://installers.edgefam.com/api/health; echo
```
